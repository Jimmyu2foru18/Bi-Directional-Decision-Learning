# 04. System Architecture

## Overview

The **Neutral-Centered Bidirectional Decision-Learning Framework (NCBDLF)** separates input representation from decision evaluation by transforming information into relative deviations from a neutral expectation state.

Instead of learning only direct correlations between inputs and outputs, the framework learns how information changes relative to:

```
𝓝₀ = Neutral Base State
```

The architecture consists of four primary stages:

1. Input normalization and neutral centering.
2. Bidirectional feature processing.
3. Decision reconstruction.
4. Dual-force parameter optimization.

---

The complete processing pipeline follows this structure:

```
Raw Input Tensor
        |
        ↓
Neutral Centering Layer
        |
        ↓
Deviation Representation (ΔX)
        |
        ↓
Bidirectional Routing Layer
        |
        ├───────────────┐
        ↓               ↓
 Positive Path       Negative Path
   S⁺                  S⁻
        ↓               ↓
        └───────┬───────┘
                ↓
      Polar Decision Mapping
                ↓
       Decision Vector Y
```

---

The first stage converts raw input values into deviations relative to a learned baseline.

Given:

```
X = input tensor
```

the framework maintains a running neutral expectation:

```
μₜ
```

The centered representation becomes:

```
X̂ₜ = Xₜ - μₜ
```

where:

- `Xₜ` = current input state.
- `μₜ` = learned neutral baseline.
- `X̂ₜ` = deviation from neutrality.

This allows the system to understand whether a signal represents:

- Positive deviation.
- Negative deviation.
- Neutral alignment.

---

After centering, the deviation representation enters two symmetrical processing paths.

Unlike traditional attention systems that create independent feature heads, NCBDLF uses mirrored structural pathways.

The two channels are:

## Positive Decision Path

```
h⁺ = NCHS(W⁺X̂ₜ)
```

Represents movement away from neutrality toward a positive decision direction.

---

## Negative Decision Path

```
h⁻ = NCHS(W⁻X̂ₜ)
```

Represents opposing movement relative to the neutral state.

Under ideal symmetry:

```
W⁻ = -W⁺
```

This allows both directions to share structural information while reducing parameter duplication.

---

The two directional states are combined to create the final decision representation.

The predicted output is:

```
Ypred = h⁺ + h⁻
```

This creates a balanced decision vector based on both:

- Attraction toward a target.
- Correction toward equilibrium.

The final output is not selected from a single pathway but emerges from the relationship between opposing states.

---

The learning algorithm follows a repeated cycle:

```
Input
 ↓
Neutral Centering
 ↓
Bidirectional Evaluation
 ↓
Decision Prediction
 ↓
Error Calculation
 ↓
Dual-Force Weight Update
 ↓
Repeat
```

---

## Forward Pass

Given an input:

```
X
```

the system performs:

1. Update neutral baseline.

2. Center the input:

```
X̂ = X - μ
```

3. Generate positive pathway:

```
h⁺ = NCHS(WX̂)
```

4. Generate negative pathway:

```
h⁻ = NCHS(-WX̂)
```

5. Combine outputs:

```
Ypred = h⁺ + h⁻
```

---

## Error Calculation

The prediction error is:

```
E = Ytarget - Ypred
```

This error determines how strongly the model should adjust.

---

## Dual-Force Optimization

The weight update contains two components:

### Learning Force

Moves weights toward useful information:

```
Gradient = E × X̂
```

---

### Neutral Restoration Force

Prevents uncontrolled growth:

```
Gravity = λ sign(W)W²
```

---

The final update:

```
W = W + η(Gradient - Gravity)
```

where:

- `η` = learning rate.
- `λ` = neutral restoration strength.

---

The following pseudocode demonstrates the core learning process:

```python
class NeutralCenteredFramework:

	def __init__(self, input_dim, output_dim):
		self.W = initialize_weights()
		self.mu = zeros(input_dim)

	def nchs(self, x):
		return tanh(alpha * x / 2)

	def forward(self, X):

		# Center input around neutral state
		X_centered = X - self.mu

		# Bidirectional processing
		h_positive = self.nchs(self.W @ X_centered)
		h_negative = self.nchs(-self.W @ X_centered)

		# Combine directional states
		Y = h_positive + h_negative

		return Y

	def train_step(self, X, target):

		prediction = self.forward(X)

		error = target - prediction

		gradient = error @ X.T

		neutral_force = (
			lambda_value *
			sign(self.W) *
			self.W**2
		)

		self.W += learning_rate * (
			gradient - neutral_force
		)
```

---

The NCBDLF architecture provides several structural benefits:

## Reduced Parameter Duplication

Because:

```
W⁻ = -W⁺
```

the negative pathway can be derived mathematically instead of stored separately.

---

## Stable Decision Movement

The neutral reference point prevents extreme movement by continuously applying restorative forces.

---

## Improved Adaptability

Because the system learns relationships around neutrality rather than fixed outcomes, it can better respond to:

- Unknown situations.
- Distribution changes.
- Limited examples.

---

The **Neutral-Centered Bidirectional Decision-Learning Framework** uses a balanced architecture where decisions emerge from the interaction between opposing pathways.

The system combines:

- Neutral-centered input processing.
- Symmetric bidirectional routing.
- Dual-force optimization.
- Restorative stability control.

This creates a learning architecture designed to remain adaptive while maintaining equilibrium around:

```
𝓝₀ = 0
```
